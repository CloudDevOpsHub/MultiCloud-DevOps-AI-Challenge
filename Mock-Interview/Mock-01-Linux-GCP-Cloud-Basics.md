## ❓ 20 Questions — Linux + GCP + Cloud Basics

### 🟢 Basic (Q1–Q5)
1. Walk me through your experience with Linux commands.
2. What is Linux and why does it matter for modern DevOps?
3. Explain the main components of Linux + GCP + Cloud Basics in simple terms.
4. What is the first thing you check if Linux fails?
5. How do you stay current on best practices for Linux + GCP + Cloud Basics?

### 🔵 Intermediate (Q6–Q10)
6. Describe a real project where you implemented Linux.
7. How do you handle security and access control in Linux + GCP + Cloud Basics?
8. Walk through setting up Linux from scratch on a new server.
9. What is your structured approach to troubleshooting Linux + GCP + Cloud Basics in production?
10. How does Linux + GCP + Cloud Basics integrate with the broader CI/CD pipeline?

### 🟡 Scenario-Based (Q11–Q15)
11. **Scenario:** Production down — logs point to Linux issue. Your action plan?
12. **Scenario:** Junior engineer misconfigured Linux + GCP + Cloud Basics. Traffic impacted. What do you do?
13. **Scenario:** Management wants 30% cost reduction in Linux + GCP + Cloud Basics infrastructure. Your approach?
14. **Scenario:** Migrate current Linux + GCP + Cloud Basics setup to new architecture with zero downtime plan.
15. **Scenario:** Security vulnerability found in your Linux + GCP + Cloud Basics pipeline. Remediation steps?

### 🔴 Senior / Architect Level (Q16–Q20)
16. Design Linux + GCP + Cloud Basics for multi-region high availability targeting 99.99% SLA.
17. What monitoring and alerting strategy do you implement for Linux + GCP + Cloud Basics?
18. Describe your disaster recovery experience involving Linux + GCP + Cloud Basics.
19. How do you mentor junior engineers on Linux + GCP + Cloud Basics best practices?
20. Top 3 mistakes teams make with Linux + GCP + Cloud Basics and how you prevent each one.
